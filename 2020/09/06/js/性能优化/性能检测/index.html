插件系列

  1. google插件FeHelper => 网页性能测

    使用：进入某一页面/刷新页面 => 点击插件下的网页性能检测，结果如下

（DOM树构建时间 = window.performance.domComplete - window.performance.domContentLoadedEventStart）

window.performance

  • 此部分参考文章

  • memory表示JS对内存的占用。

  • navigation：导航相关数据

  • timing：最重要的字段，包含了网络、解析等一系列的时间数据

    • domLoading浏览器开始解析html文档的时间节点。其值就等于readyState变为loading的时间节点；

    • domContentLoadedEventStart：DOMContentLoaded事件触发的时间节点

      • DOM完全加载并解析完毕之后,会触发DOMContentLoaded事件，但不会等待样式文件,图片文件,子框架页面的加载

    • domContentLoadedEventEnd：DOMContentLoaded事件完成的时间节点，此时用户可以对页面进行操作，也就是jQuery中的domready时间

    • domComplete：html文档完全解析完毕的时间节点；

    • loadEventStart和loadEventEnd分别代表onload事件触发和结束的时间节点

    • requestStart代表浏览器发起请求的时间节点，请求的方式可以是请求服务器、缓存、本地资源等；

    • responseStart和responseEnd分别代表浏览器收到从服务器端（或缓存、本地资源）响应回的第一个字节和最后一个字节数据的时刻

  • 通过timing计算性能指标
    • DNS查询耗时 = domainLookupEnd - domainLookupStart
    • TCP链接耗时 = connectEnd - connectStart
    • request请求耗时 = responseEnd - responseStart
    • 解析dom树耗时 = domComplete - domInteractive
    • 白屏时间 = domloadng - fetchStart
    • domready时间 = domContentLoadedEventEnd - fetchStart
    • onload时间 = loadEventEnd - fetchStart
