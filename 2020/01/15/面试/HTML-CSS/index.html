HTML

说说HTML5在标签、属性、存储、API上的新特性

  • 标签：新增语义化标签（aside / figure / section / header / footer / nav等），增加多媒体标签video和audio，使得样式和结构更加分离
  • 属性：增强表单，主要是增强了input的type属性；meta增加charset以设置字符集；script增加async以异步加载脚本
  • 存储：增加localStorage、sessionStorage和indexedDB，引入了application cache对web和应用进行缓存
  • API：增加拖放API、地理定位、SVG绘图、canvas绘图、Web Worker、WebSocket

DocType是干什么的

<!DOCTYPE>用于告知浏览器使用哪种HTML或XHTML规范来渲染文档

  • BackCompat：怪异模式，浏览器使用自己的怪异模式解析渲染页面。（默认）
  • CSS1Compat：标准模式，浏览器使用W3C的标准解析渲染页面。

渐进增强和优雅降级

渐进增强：针对低版本浏览器进行构建页面，保证最基本的功能，然后再针对高级浏览器进行效果、交互等改进和追加功能达到更好的用户体验。

优雅降级：一开始就构建完整的功能，然后再针对低版本浏览器进行兼容

src与href的区别

  • src：指向外部资源的位置，用于替换当前元素
    • 当浏览器解析到该元素时，会暂停其他资源的下载和处理，直到将该资源加载、编译、执行完毕
  • href：用于在当前文档和引用资源之间确立联系
    • 当浏览器解析到该元素时，会并行下载资源并且不会停止对当前文档的处理

img标签的srcset的作用是什么

可以设计响应式图片，我们可以使用srcset和sizes来提供更多额外的资源图像和提示，帮助浏览器选择正确的一个资源

  • srcset用于 浏览器根据宽、高和像素密度来加载相应的图片资源。

  • sizes用于给图片设置一个预估的显示宽度

还有哪一个标签能起到跟srcset相似作用？

<picture>元素通过包含零或多个 source 元素和一个img 元素来为不同的显示/设备场景提供图像版本。浏览器会选择最匹配的子 source 元素，如果没有匹配的，就选择 img 元素的 src 属性中的URL。然后，所选图像呈现在<img>元素占据的空间中

script标签中的defer和async的区别

  • defer：脚本在文档被解析后执行，script被异步加载后并不会立刻执行，而是等待文档被解析完毕后执行。

  • async：同样是异步加载脚本，区别是脚本加载完毕后立即执行，这会导致脚本乱序，对于script有先后依赖关系的情况，并不适用。

蓝色线代表网络读取，红色线代表执行时间，这俩都是针对脚本的；绿色线代表 HTML 解析

meta有哪些属性，作用是什么

meta标签用于描述网页的元信息，如网站作者、描述、关键词，meta通过name=xxx和content=xxx的形式来定义信息

  • charset：定义HTML文档的字符集

  • http-equiv：可用于模拟http请求头，可设置过期时间、缓存、刷新

  • viewport：视口，用于控制页面宽高及缩放比例

viewport有哪些参数

  • width/height，宽高，默认宽度980px
  • initial-scale，初始缩放比例，1~10
  • maximum-scale/minimum-scale，允许用户缩放的最大/小比例
  • user-scalable，用户是否可以缩放 (yes/no)

http-equive属性的作用和参数

  • expires，指定过期时间
  • progma，设置no-cache可以禁止缓存
  • refresh，定时刷新
  • set-cookie，可以设置cookie
  • X-UA-Compatible，使用浏览器版本
  • apple-mobile-web-app-status-bar-style，针对WebApp全屏模式，隐藏状态栏/设置状态栏颜色

事件流

事件流又称为事件传播，DOM2级事件规定的事件流包括三个阶段：事件捕获阶段(capture phase)、处于目标阶段(target phase)和事件冒泡阶段(bubbling phase)。

  1. 捕获阶段:：从最外层的祖先元素,向目标元素进行事件的捕获,但是默认会不会触发事件
  2. 目标阶段：事件捕获到目标元素,开始在目标元素上触发事件
  3. 冒泡阶段：事件从目标元素向祖先元素传递,依次触发祖先元素上的事件

  • 事件委托： 利用事件冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件

    • 优点：

      • 节省内存，减少事件注册
      • 新增子对象时无需紫安琪对其绑定事件，适合动态添加元素

    • 局限性：

      • focus、blur 之类的事件本身没有事件冒泡机制，所以无法委托
      • mousemove、mouseout 这样的事件，虽然有事件冒泡，但是只能不断通过位置去计算定位，对性能消耗高，不适合事件委托

rgba和opacity的透明效果

  • rgba：只作用于元素的颜色或背景色
  • opacity：作用于元素，以及元素内所有内容的透明度

为什么要初始化样式？

由于浏览器兼容的问题，不同的浏览器对标签的默认样式值不同，若不初始化则会造成不同浏览器之间的显示差异

BFC是什么

  • BFC：块级格式化上下文

    ​ 它是一个独立的渲染区域，该区域内所有元素的布局不会影响到区域外部元素的布局，这个渲染区域只对块级元素起作用。

  • BFC渲染规则

    • BFC是页面上的一个隔离的独立容器，不受外界干扰或干扰外界
    • BFC区域不会与浮动的元素重叠
    • 计算BFC高度时,浮动元素也参与计算
    • BFC内部两个相邻元素的margin会发生重叠
    • BFC内部的元素会在垂直方向上放置

  • 如何开启BFC

    • float: left/right
    • position: absolute/fixed
    • display: inline-block/table-cell/flex/inline-flex
    • overflow: hidden/auto/scroll
      • ie6: zoom: 1

  • 应用

    • 自适应两栏布局（2）

    • 清除内部浮动

    • 防止垂直margin重叠 （1）

      折叠结果遵循下列计算规则：

      1. 两个外边距都是正数时，结果是两者较大的值。

      2. 两个外边距都是负数时，结果是两者绝对值的较大值。

      3. 两个外边距一正一负时，结果是两者的相加的和。

语义化的好处

  • 语义化：使用标签来表示网页中的不同组成部分
  • 好处
    1. 提升搜索引擎优化（SEO）的效果
    2. 使维护代码和添加样式变得容易。
    3. （通常）使代码更少，页面加载更快

querySelectorAll与getElementsBy

  • querySelectorAll与getElementsBy系列区别

    1. querySelectorAll属于W3C中的Selectors API规范,而getElementsBy系列属于W3C DOM规范
    2. querySelectorAll方法接受参数是CSS选择符,当传入字符串不符合CSS选择符规范时会抛出异常,getElementsBy系列只接受单一的参数
    3. querySelectorAll返回的是不变(静态)的节点列表,getElementsBy系列返回的是动态节点
    4. getElementsBy性能较好
    5. querySelectorAll返回值是一个NodeList, getElementsBy返回值是一个HTMLCollection

  • HTMLCollection与NodeList区别

    1. HTMLCollection是元素节点集合,而NodeList是节点集合(包含任何类型的节点)
    2. node.children 和 node.getElementsByXXX 返回 HTMLCollection，而node.childNodes，querySelectorAll返回的是 NodeList 。
