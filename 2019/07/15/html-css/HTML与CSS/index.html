一.HTML

1.元素

重点在于语义，而非样式
使用标签来标识网页中的不同组成部分
.
语义化的好处

  • 提升可访问性和互操作性（内容对于借助辅助技术的残障访问者是可访问的， 同时对于台式机、手机、平板电脑及其他设备上的浏览器都是可访问的）。
  • 提升搜索引擎优化（SEO）的效果。
  • 使维护代码和添加样式变得容易。
  • （通常）使代码更少，页面加载更快。

1.1 元信息标签

  • head：不携带任何信息，只是盛放其他语义类标签的容器
  • title：标题标签
  • base：指定a标签的默认窗口 <base target="_blank" >
  • meta
    一组键值对
    四个属性: name http-equiv content charset
    charset: 定义文档的解码方式(编码方式由编辑器决定)
    name: 与content组合,页面文档的元信息
  1. “viewport” content=”键=值,键=值…”移动设备优化
    (width/height=数字/device-width/height initial/minimum/maximum-scale user-scalable)
  2. “author”页面作者
  3. “description” 页面描述,可能用于搜索引擎
  4. name=”generator” 生成页面所使用的工具,主要用于可视化编辑器。(则不需要加这个)
  5. “keywords” 对于SEO场景十分关键(1,2,3)
  6. name=”referrer” 跳转策略,是一种安全考量
  7. name=”theme-color” 页面风格颜色,实际上并不会影响页面,但浏览器可能据此调整页面之外的UI（如窗口边框 或者tab的颜色）。

http-equiv: 与content组合,设置HTTP请求指令

  1. http-equiv=”content-type” content=”text/html;charset=utf-8”
  2. http-equiv=”content-language” 指定内容的语言
  3. http-equiv=”dafault-style” 指定默认样式表(很少用)
  4. http-equiv=”refresh” 刷新(不推荐使用,某些搜索引擎meta遇到此会停止解析页面剩余部分)
  5. http-equiv=”set-cookie” 设置cookie
  6. http-equiv=”x-ua-compatible” 设置IE浏览器的兼容模式(content=”ie-8”)
  7. http-equiv=”content-security-policy” 声明内容安全策略

1.3.2 a
  • anchor: 文档中的特定位置
  • 属性:
    rel=alternate/author/hel/license/next/prev/search
    rel=tag: 本网页所属标签
    rel=bookmark: 链接到上级章节
    rel=nofollow: 此链接不会被搜索引擎索引
    rel=noopener: 此链接打开的网页无法使用opener来获得当前页面的窗口
    rel=noreferrer: 此链接打开的网页无法使用referrer来获得当前页面的url
    rel=opener: 打开的网页可以使用window.opener来访问当前页面的window对象，这是a标签的默认行为。

1.3 单位

移动端单位
  • 物理像素(phisical pixel): pp
  • 设备独立像素(density-independent pixel dip): 密度无关像素(设备出厂固定) ipnone5 640px / iphone6 750px
  • CSS像素(device-independent pixel): 设备无关像素DIPs
    主要使用在浏览器上，用来精确度量Web页面上的内容。
  • 屏幕密度: 每英寸的像素数(PPI)。
  • 设备像素比(dpr): 逻辑像素 / 物理像素
    ipnone5/6 : 2 ipad : 3
  • vw / vh 视口宽度/高度
  • vmin：vw和vh中较小的那个。vmax：vw和vh中较大的那个。

媒体类型
  1. all: 所有设备
  2. screen:电脑显示器
  3. print: 打印用纸或打印预览视图
  4. braille: 盲人用点字法触觉回馈设备
  5. embossed: 盲文打印机
  6. handheld: 手持设备
  7. projection: 投影设备
  8. speech: 语音或音频合成器
