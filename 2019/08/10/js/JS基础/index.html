JS知识点扫盲

Aug 10, 2019

JavaScript

  • 一个完整的JavaScript实现应该由以下三个部分构成
    1. ECMAScript
    2. DOM(文档对象模型)
    3. BOM(浏览器对象模型)

数据类型

  • 基本数据类型

    1. String

      a. 引号必须在同一行
      b. 引号不可嵌套

    2. Number

      • tofixed(n) 返回小数形式的字符串(n为小数位数)
      • toExceptional(n) 返回指数形式的字符串(n为小数位数)
      • toPrecisionn为有效数字位数
      • toString(radix)返回指定基数形式的字符串

    3. Boolean

    4. Null：typeof null === "object"

    5. Undefined：typeof undefined === "undefined"

  • 引用数据类型

    1. Object

  • 为了便于操作基本类型值，ES提供了3种特殊的引用类型

    1. String
    2. Number
    3. Boolean

    • 每当基本类型值调用引用类型所属的方法时,后台会自动完成下列处理

      1. 创建一个相应的引用类型的实例
      2. 在这个实例上调用指定的方法
      3. 销毁这个实例
      • 这意味着：不能再运行时为基本类型值添加属性和方法

数据类型转换

  • 转String

    1. xx.toString
      • null和undefined这两个值没有toString()方法
    2. String(xx)
      • 对Number和Boolean,实际上是调用toString()方法
      • 对于null和undefined,就不会调用toString(),而是直接转成"null"或"undefined"
    3. xx + ""

  • 转Number

    1. Number()
    2. 字符串转数字：parseInt(str, radix)和parseFloat
    3. xx-0 xx*1 xx/1
    4. +xx正号而非加号

  • 转Boolean

    1. Boolean()
    2. !!xx

void

  • 流程:

    1. 令expr为解释执行的结果
    2. 调用getValue(expr)
    3. 返回undefined

  • 为什么 要用void
    因为undefined在js中不是保留字,因而可以作为变量名被赋值,所以不是绝对可靠的而void能保证渠道undefined值

  • 作用

    1. void 0取undefined
    2. 填充<a>的href确保点击不会发生页面跳转
    3. 填充<img>的src,确保不会向服务器发出垃圾请求
