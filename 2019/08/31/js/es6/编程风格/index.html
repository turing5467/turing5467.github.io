ES6 编程风格

Aug 31, 2019

ES6编程风格

  1. let const取代var

  2. 字符串一律使用单引号''或反引号

  3. 使用数组对变量赋值时，优先使用解构赋值

    函数参数/返回值如果是对象的成员，，优先使用解构赋值

对象

  1. 单行对象，最后一个成员不以,结尾

    多行对象，最后一个成员以,结尾

  2. 对象尽量静态化，一旦定义，就不得随意添加新的属性。

    如果添加属性不可避免，要使用 Object.assign 方法。

  3. 对象的属性和方法，尽量采用简洁表达法

数组

  1. 使用扩展运算符拷贝数组
  2. 使用Array.from，将类数组转为数组

函数

  1. 立即执行函数写出箭头函数的形式.

  2. 需要函数表达式的场合，尽量用箭头函数代替。

  3. 箭头函数取代bind，不应再用_self _this that绑定this

  4. 简单的、单行的、不会复用的函数，建议采用箭头函数。

    如果函数体较为复杂，行数较多，还是应该采用传统的函数写法。

  5. 所有配置项都应该集中在一个对象，放在最后一个参数，布尔值不可以直接作为参数。

  6. 不要在函数体内使用arguments变量，使用rest元素运算符...代替

  7. 使用默认值语法设置函数参数的默认值。

模块

  1. import取代require
  2. export取代module.exports
  3. 如果模块只有一个输出值，就使用 export default
  4. 如果模块默认输出一个函数，函数名的首字母小写。
  5. 如果模块默认输出一个对象，对象名的首字母大写。

class

  1. 总是用class，取代需要prototype的操作。

  2. 使用 extends 实现继承

Map结构

  • 注意区分Object和Map，只有模拟现实世界的实体对象时，才使用Object。

    如果只是需要 key: value 的数据结构，使用Map结构。因为Map有内建的遍历机制。
