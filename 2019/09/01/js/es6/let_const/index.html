ES6----let、const

Sep 1, 2019

let

  1. 块级作用域内有效

  2. 不存在变量提升

    凡是在声明之前就使用 let声明的变量 ，就会报错(暂时性死区TDZ)

  3. 不允许重复声明

  4. 外层代码块不受内层代码块的影响。

const

  1. 一旦声明，就不能改变，且必须立即初始化
  2. 块级作用域有效
  3. 不存在变量提升
  4. 对于复合类型的变量，变量指向数据所在的地址。 const 命令只保证变量名指向的地址不变。

  • 如果真想将对象冻结，应该使用Object.freeze()方法

  • 除了将对象本身冻结，对象类型的属性也应该冻结。下面是将对象彻底冻结的函数。

声明变量の方式

ES5：var、function

ES6：var、function、let、const、import、class

顶层对象的属性

顶层对象，在浏览器环境指的是 window 对象，在Node指的是 global 对象。

ES5：顶层对象的属性与全局变量是等价的。

ES6：

  1. var 命令和 function 命令声明的全局变量，依旧是顶层对象的属性；

  2. let 命令、 const 命令、 class 命令声明的全局变量，不属于顶层对象的属性。
