typescript入门

Oct 15, 2019

基础类型

  1. boolean

  2. number

  3. string

  4. 数组：Array<number> 或number[]

  5. 元组类型允许表示一个已知元素数量和类型的数组，各元素的类型可以不同

  6. 枚举(enum：可以定义一些有名字的数字常量，索引默认从0递增

    可以从枚举值到枚举名进行反向映射。

  1. any 表示任何类型

  2. void 表示没有任何类型,只能赋予null和undefined，一般用于无返回值的函数

  3. null 和undefined

  4. never 永不存在的值的类型，

    例如 throw new Error

    ​ return error('sth failed')

    ​ while(true) {...}

  • 类型断言assert：可以用来手动指定一个值的类型

    1. 尖括号语法：<string>variable

    2. as语法：variable as string

      当在TypeScript里使用JSX时，只有as语法断言是被允许的。

    • 场景：在还不确定类型的时候 就得访问其中一个类型的属性或方法

  • 泛型：函数中的变量的类型与传入参数的类型相同

    • 泛型约束

      • extends 

- `keyof` 索引类型查询操作符。

  `keyof T`产生的类型是T的**属性所属类型**构成的**联合类型**。

  • 函数

  • 函数可选参数：参数后跟?

    注意：可选参数必须位于参数列表的最后。

  • 参数默认值：形参名称 : 参数类型 = 默认值

自定义类型

1. `interface`
 2. `type`

  • 相同点：

    1. 都可以用来描述一个对象或函数

  1. 都允许继承，且可相互继承

  • 不同点

    1. type可以而interface不可以
      • 声明基本类型别名，联合类型，元组等类型
      • 可以使用typeof获取实例的类型进行赋值
      • | keyof等
    2. interface可以而type不行
      • 声明合并
      • 可选属性：后跟?
      • 只读属性：前用readonly

实现与继承

  1. implements 明确的强制一个类去符合某种契约
  2. extends 继承父类

访问修饰符

  • public默认
  • private 无法被外部访问
  • protected只能在子类中访问
