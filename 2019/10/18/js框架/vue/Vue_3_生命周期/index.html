Oct 18, 2019

Vue 的生命周期

  • 三大阶段

    1. 初始化：mounted之前
    2. 更新：
    3. 死亡：vm.$destroy()

  • 常用声明周期方法

    1. mounted()初始化显示之后立即调用(1次)

      □ 发送ajax请求，启动定时器等异步任务

    2. beforeDestroy：实例销毁之前调用，这一步时，实例还是可用的

      □ 做收尾工作，如清除定时器

生命周期钩子函数

  • 创建前后
    • beforeCreate
    • created
  • 挂载
    • beforeMount 如果有el属性，则$el可用，但数据还未渲染
    • mounted 处理template
  • 更新
    • beforeUpdate
    • updated
  • 激活 keep-alive：销毁真实DOM,保留虚拟DM，
    • activated 组件是否被激活
    • dectivated
  • 销毁：真实DOM和虚拟DOM都会销毁，如componnet :is或v-if都会触发激活事件
    • 可用来关闭定时器
    • beforeDestory
    • destroyed
  • 监听路由错误
    • errorCaptured 当捕获一个来自子孙组件的错误时被调用
      • 参数：(错误对象，发生错误的组件实例， 错误信息字符串)

注意点，Vue的所有生命周期函数都是自动绑定到this的上下文上。

编译模板优先级：render函数 > 内部template > 外部 html

钩子 详细介绍（ 完成 后执行 钩子函数）
beforeCreate 实例在内存中被创建，初始化事件和生命周期
created 初始化 props,methods ,data,computed 和 watch
beforeMount 检查是否存在 el ，存在的话编译模板，render => 生成虚拟DOM
mounted 挂载点将虚拟DOM渲染为真实DOM
beforeUpdate 实例上的数据已经更新，但是还没有同步到页面上
updated 数据已同步至页面，
检查当前的 watcher 列表中，是否存在当前要更新数据的 watcher ，如果存在就执行 updated ；
beforeDestroy 检查是否已经被卸载，如果已经被卸载，就直接 return 出去，否则执行 beforeDestroy ；
destroyed 把所有有关自己痕迹的地方，都给删除掉；

Vue.nextTick

  • 数据层变化的时候，并不会立即更新真实DOM，它需要一个差异对比的更新过程

  • 为了让数据完全渲染到视图后再去触发逻辑，我们可以使用Vue.nextTick()函数

    • Vue.nextTick实现原理其实就是setTimeout()
